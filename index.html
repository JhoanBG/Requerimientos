Requerimentos Funcionales y no Funcionales del Software de automatización de pedidos para el restaurante de comidas rapidas Comic Hamburguer
Requerimentos Funcionales:
  • El sistema debe permitir llevar un registro de mesas disponibles y no disponibles.
  • El sistema debe asginar un número de mesa al cliente, dependiendo de la disponibilidad de las mismas.
  • El sistema debe permitir la actualización fácil y rápida del menú, incluyendo platillos, precios y descripciones.
  • El sistema debe permitir al cliente registrar su pedido de forma eficiente.
  • El sistema debe calcular automáticamente el total de la factura en base a los pedidos realizados.
  • El sistema debe permitir al cliente realizar pagos electrónicos de manera eficiente.
  • El sistema debe realizar un seguimiento automático del inventario de ingredientes y productos para evitar situaciones de falta de existencias.
  • El sistema debe permitir la creación y administración de cuentas de usuario para el personal del restaurante con diferentes niveles de acceso (meseros, cocineros, administradores).

    • Requerimentos no Funcionales:
  • El sistema debe responder de manera rápida y eficiente para garantizar una experiencia de usuario ágil.
  • El sistema debe implementar medidas de seguridad robustas para proteger la información confidencial, como datos de clientes y registros financieros.
  • El sistema debe ser escalable para adaptarse al crecimiento del restaurante, permitiendo la incorporación de nuevos platillos, personal y funciones sin comprometer el rendimiento.
  • La interfaz de usuario debe ser intuitiva y fácil de usar, incluso para el personal que no esté familiarizado con sistemas informáticos complejos.
  • El sistema debe estar disponible durante las horas de operación del restaurante.
  • El sistema debe ser capaz de manejar un número significativo de transacciones simultáneas durante las horas pico sin experimentar una degradación significativa del rendimiento.
  • El sistema debe ser fácil de mantener y actualizar para incorporar nuevas características o corregir posibles problemas sin interrumpir las operaciones diarias.
